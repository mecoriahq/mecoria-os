# Hiddenova Graphic Quality Standard v1

## Purpose

Hiddenova visuals must feel like premium documentary graphics, not basic presentation slides.

This standard applies to:
- diagrams
- map graphics
- text overlays
- motion graphics
- internal visual assets
- video assembly graphics

The goal is to move from technical placeholder visuals to public-quality YouTube documentary visuals.

---

## Core Visual Direction

Hiddenova graphics should feel:
- cinematic
- modern
- clean
- investigative
- premium
- calm but engaging
- explanatory, not decorative

Avoid:
- PowerPoint-style diagrams
- flat template visuals
- generic AI slideshow look
- cluttered dashboards
- fake UI screens
- fake labels
- fake barcodes
- fake brands
- excessive text

---

## Color System

Use a restrained documentary palette.

Primary palette:
- Background: deep charcoal / near black
- Panels: dark blue-gray
- Main flow: muted electric blue
- Accent: warm amber
- Positive route: muted green
- Warning route: muted red
- Main text: soft white
- Secondary text: muted gray-blue

Rules:
- Do not use bright toy-like colors.
- Do not use more than two accent colors in one graphic.
- Blue explains flow.
- Amber highlights attention.
- Green and red are only for decision states.
- Backgrounds should have subtle depth through gradients or texture.

---

## Typography

Typography should feel editorial and premium.

Rules:
- Use strong title hierarchy.
- Titles should be clear but not oversized.
- Use short labels.
- Avoid long text inside boxes.
- Break long concepts into smaller nodes.

Prefer:
- YES - RESTOCK
- NO - INSPECT
- REPAIR
- RESELL
- LIQUIDATE

Avoid:
- long cramped labels
- all-caps paragraphs
- text touching box edges
- too much footer text

---

## Diagram Rules

A diagram must explain one idea clearly.

Good diagram structure:
- one hero decision or concept
- two to four primary branches
- clear hierarchy
- balanced spacing
- short labels
- enough breathing room
- clean node alignment

Avoid:
- uneven layouts
- cramped nodes
- long horizontal labels
- random line angles
- too much empty space
- too many concepts in one frame

Example improvement:

Instead of:
NO - INSPECT
down to:
REPAIR / RESELL / LIQUIDATE

Prefer:
NO - INSPECT
down to three separate nodes:
REPAIR
RESELL
LIQUIDATE

This creates better visual rhythm and avoids cramped text.

---

## Map / Network Graphic Rules

Map graphics should be abstract, not literal.

Use:
- home node
- carrier hub
- return center
- inspection node
- refurbisher
- liquidator
- reseller
- recycler
- new customer

Avoid:
- real street grids
- exact routes
- addresses
- company names
- fake tracking screens
- fake UI
- fake location labels

Map graphics should show:
- one item entering the network
- branching routes
- convergence into hubs
- recommerce loop
- second-life path

---

## Text Overlay Rules

Text overlays must be minimal and cinematic.

Use:
- short headline
- one supporting line
- subtle motion or fade
- strong negative space

Good examples:
- WHAT HAPPENS NOW?
- CAN THIS PRODUCT BEGIN AGAIN?
- THE HIDDEN SYSTEMS BEHIND EVERYDAY LIFE

Avoid:
- paragraph overlays
- busy UI graphics
- excessive brand marks
- decorative icons with no purpose

---

## Motion Graphic Rules

Motion graphics should clarify systems.

Use motion for:
- route reveal
- decision branch reveal
- value decay
- cost accumulation
- loop formation
- node highlighting

Avoid:
- movement only for decoration
- flashy transitions
- overcomplicated animations
- fake dashboards
- unsupported numbers

---

## Composition Rules

Every graphic must pass these checks:
- Is the main idea clear in two seconds?
- Is the layout balanced?
- Are labels short and readable?
- Is there enough spacing?
- Does it feel cinematic, not like a slide?
- Could this appear in a serious YouTube documentary?

If not, revise before video assembly.

---

## Quality Levels

Level 0 - Technical Placeholder:
- readable
- functional
- basic
- acceptable only for system testing

Level 1 - Internal Draft:
- clean layout
- consistent colors
- understandable
- not public quality yet

Level 2 - Public MVP:
- modern layout
- good spacing
- strong hierarchy
- cinematic background
- usable in unlisted or public test

Level 3 - Premium Hiddenova:
- documentary-grade
- animated
- visually memorable
- consistent across the whole video
- strong enough for public release

Current target:
Level 2 first, Level 3 later.

---

## Immediate Improvement Targets

The first internal visual render proved that the system works, but the quality is too basic.

Required improvements:
1. Replace flat box diagrams with modern node/card layouts.
2. Break long labels into multiple smaller nodes.
3. Improve spacing and visual balance.
4. Add subtle depth, glow, and panel hierarchy.
5. Use better visual rhythm.
6. Make diagrams feel like documentary graphics, not presentation slides.
7. Preserve readability.
8. Avoid overdecorating.

---

## Rule for Agents

Agents must not optimize only for technical correctness.

They must optimize for:
- clarity
- documentary feel
- modern composition
- reuse
- readability
- public-quality direction

A graphic can be technically valid and still be rejected for being too basic.
